# Specification: User Registration & Login (FEAT-001)

## Functional Requirements
- يجب أن يستطيع المستخدم التسجيل باستخدام البريد الإلكتروني وكلمة مرور.
- يجب أن يختار المستخدم نوع الحساب (مشارك أو منظم) أثناء التسجيل.
- يجب أن يستطيع المستخدم تسجيل الدخول باستخدام بياناته.
- يجب أن تظهر رسائل خطأ واضحة عند إدخال بيانات غير صحيحة.
- يجب أن تكون جميع الواجهات باللغة العربية.

## Non-Functional Requirements
- سرعة الاستجابة في واجهات التسجيل والدخول.
- دعم كامل للهواتف المحمولة.
- واجهة بسيطة وسهلة الاستخدام.

## Data Model
- User:
  - Id
  - Email
  - PasswordHash
  - AccountType (Participant, Organizer)
  - CreatedAt

## API Endpoints
- POST /api/account/register
  - البيانات: البريد الإلكتروني، كلمة المرور، نوع الحساب
- POST /api/account/login
  - البيانات: البريد الإلكتروني، كلمة المرور

## UI Components
- صفحة التسجيل:
  - نموذج إدخال البريد الإلكتروني
  - نموذج إدخال كلمة المرور
  - اختيار نوع الحساب (قائمة منسدلة)
  - زر التسجيل
  - رسائل الخطأ
- صفحة تسجيل الدخول:
  - نموذج إدخال البريد الإلكتروني
  - نموذج إدخال كلمة المرور
  - زر الدخول
  - رسائل الخطأ

## Validation
- تحقق من صحة البريد الإلكتروني وصيغة كلمة المرور.
- منع التسجيل ببريد إلكتروني مستخدم سابقًا.
